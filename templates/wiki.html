    Linux-Installation (Ubuntu / Debian)

    Empfohlen: Ubuntu 22.04 LTS oder Debian 12. Mindestanforderungen: 1 vCPU, 512 MB RAM, 5 GB Disk.

    1. System aktualisieren & Python installieren

    sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-pip python3-venv git

    2. Benutzer anlegen (optional, empfohlen)

    sudo useradd -m -s /bin/bash itassets
sudo su - itassets

    3. Anwendung herunterladen

    # Als itassets-Benutzer (oder in /opt/itassets)
git clone https://github.com/bohlerdavid/web.git /opt/itassets
cd /opt/itassets

    4. Python Virtual Environment erstellen

    python3 -m venv venv
source venv/bin/activate

    5. Python-Pakete installieren

    pip install --upgrade pip
pip install -r requirements.txt
pip install gunicorn

    Inhalt der requirements.txt:

    Flask==3.0.3
Werkzeug==3.0.3
python-nmap
paramiko
    Hinweis: Das wmi-Paket ist nur unter Windows verfügbar. Auf Linux wird ausschliesslich SSH für Hardware-Abfragen verwendet.

    6. Datenbank initialisieren & ersten Admin anlegen

    source venv/bin/activate
python create_user.py
# Benutzername: admin
# Vollständiger Name: Administrator
# Passwort: (sicheres Passwort wählen)
# Rolle: admin

    Nginx + Gunicorn (Produktionsbetrieb)

    1. Nginx installieren

    sudo apt install -y nginx

    2. Gunicorn testen

    cd /opt/itassets
source venv/bin/activate
gunicorn --bind 127.0.0.1:5000 app:app

    3. Nginx-Konfiguration erstellen

    sudo nano /etc/nginx/sites-available/itassets

    Inhalt der Konfigurationsdatei:

    server {
    listen 80;
    server_name DEINE-IP-ODER-DOMAIN;   # z. B. 192.168.1.50 oder it.firma.local

    client_max_body_size 10M;

    location /static/ {
        alias /opt/itassets/static/;
        expires 7d;
        add_header Cache-Control "public";
    }

    location / {
        proxy_pass         http://127.0.0.1:5000;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }
}

    4. Site aktivieren

    sudo ln -s /etc/nginx/sites-available/itassets /etc/nginx/sites-enabled/
sudo nginx -t        # Konfiguration testen
sudo systemctl reload nginx
    HTTPS empfohlen: Mit sudo apt install certbot python3-certbot-nginx und sudo certbot --nginx -d deine-domain.ch kann Let's Encrypt kostenlos eingerichtet werden (nur bei öffentlicher Domain).

    Systemd Service (Autostart)

    1. Service-Datei erstellen

    sudo nano /etc/systemd/system/itassets.service

    Inhalt:

    [Unit]
Description=IT Assets Verwaltung (Flask/Gunicorn)
After=network.target

[Service]
User=itassets
Group=www-data
WorkingDirectory=/opt/itassets
Environment="PATH=/opt/itassets/venv/bin"
Environment="SECRET_KEY=AENDERN-AUF-LANGEN-ZUFAELLIGEN-STRING"
ExecStart=/opt/itassets/venv/bin/gunicorn \
    --workers 2 \
    --bind 127.0.0.1:5000 \
    --timeout 120 \
    --access-logfile /var/log/itassets/access.log \
    --error-logfile /var/log/itassets/error.log \
    app:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

    2. Log-Verzeichnis erstellen & Service starten

    sudo mkdir -p /var/log/itassets
sudo chown itassets:www-data /var/log/itassets

sudo systemctl daemon-reload
sudo systemctl enable itassets
sudo systemctl start itassets
sudo systemctl status itassets

    Nützliche Service-Befehle

    # Status anzeigen
sudo systemctl status itassets

# Logs anzeigen (live)
sudo journalctl -u itassets -f

# Neustart
sudo systemctl restart itassets

# Stoppen
sudo systemctl stop itassets

    Netzwerk-Scanner Tools installieren

    Alle drei Tools werden vom Netzwerkscanner automatisch erkannt. Je mehr installiert sind, desto besser die Scan-Ergebnisse.

    nmap (OS-Erkennung, Ports)

    sudo apt install -y nmap

    arp-scan (schnelle MAC-Erkennung im LAN)

    sudo apt install -y arp-scan

# Dem itassets-Benutzer erlauben arp-scan ohne sudo auszuführen:
sudo setcap cap_net_raw+ep /usr/sbin/arp-scan

    masscan (schneller Port-Scanner für grosse Netzwerke)

    sudo apt install -y masscan

# Dem itassets-Benutzer erlauben:
sudo setcap cap_net_raw+ep /usr/bin/masscan

    SSH-Zugriff für Hardware-Abfragen (Linux-Clients)

    Der Server verbindet sich via SSH auf Linux-Clients um CPU, RAM und Disk-Daten abzufragen. Dafür braucht es:

    • SSH-Zugang mit Benutzername/Passwort (im Scanner-Tab eingebbar)
    • Auf dem Ziel-Client: sudo apt install -y openssh-server
    • Für dmidecode ohne sudo: sudo chmod +s /usr/sbin/dmidecode

    Firewall (UFW)

    sudo apt install -y ufw

# Wichtige Ports freigeben
sudo ufw allow OpenSSH        # SSH (Port 22)
sudo ufw allow 80/tcp         # HTTP (Nginx)
sudo ufw allow 443/tcp        # HTTPS (optional, für SSL)

# Nur aus dem internen Netz (empfohlen):
# sudo ufw allow from 192.168.1.0/24 to any port 80

sudo ufw enable
sudo ufw status
    Die Anwendung sollte nicht direkt aus dem Internet erreichbar sein — nur im internen Firmennetz oder via VPN.

    Erststart & Konfiguration

    1
    Browser öffnen und http://SERVER-IP aufrufen
    2
    Mit dem zuvor erstellten Admin-Benutzer anmelden
    3
    Unter Standorte die Gebäude/Räume anlegen
    4
    Unter Layout (Admin) eigene Sektionen und Felder für die Gerätdetailseite definieren
    5
    Ersten Netzwerkscan unter Netzwerkscan starten und Geräte importieren
    6
    Benutzer für Kollegen unter Benutzer anlegen (Rolle: Betrachter für Nur-Lesen)

    Sicherheits-Checkliste

    • Standard-Admin-Passwort sofort ändern
    • SECRET_KEY in der systemd-Umgebungsvariable auf einen langen Zufallswert setzen: python3 -c "import secrets; print(secrets.token_hex(32))"
    • Zugriff auf die Anwendung auf das interne Netz beschränken (Firewall)
    • Regelmässige Backups der devices.db einrichten

    Datenbank-Backup

    # Manuell
cp /opt/itassets/devices.db /backup/devices_$(date +%Y%m%d).db

# Automatisch täglich mit Cron (als root):
echo "0 2 * * * cp /opt/itassets/devices.db /backup/devices_\$(date +\%Y\%m\%d).db" | sudo crontab -

    Anwendung aktualisieren

    cd /opt/itassets

# 1. Backup der Datenbank
cp devices.db devices_backup_$(date +%Y%m%d).db

# 2. Neueste Version laden
git pull origin claude/device-management-system-uQhWP

# 3. Abhängigkeiten aktualisieren
source venv/bin/activate
pip install -r requirements.txt

# 4. Service neustarten
sudo systemctl restart itassets
    Die Datenbank wird beim Start automatisch migriert — bestehende Daten bleiben erhalten.

    Windows (Entwicklung / lokaler Test)

    Voraussetzungen

    • Python 3.10+ von python.org (mit "Add to PATH" installieren)
    • Git von git-scm.com
    • Nmap von nmap.org (WinPcap mitinstallieren)

    Installation

    git clone https://github.com/bohlerdavid/web.git it-verwaltung
cd it-verwaltung
pip install -r requirements.txt
pip install wmi pywin32 paramiko

    Starten

    python app.py

    Danach im Browser: http://localhost:5000

    Das wmi-Paket (Windows-Hardware-Abfragen) ist nur unter Windows verfügbar und wird auf Linux automatisch übersprungen.
